# Project Name

## 🎯 Описание

Набор мини-игр, реализованных на базе одного универсального класса «шарика». 
Проекты используют общую библиотеку и переопределяют поведение объектов для создания различных игровых сценариев: лопание шариков, бильярд,
симуляция газов, салюты, а также пародийные версии Fruit Ninja и Angry Birds.
Репозиторий демонстрирует, как единая абстракция может применяться в разных задачах — от простой анимации до физического моделирования.


Сделал этот проект, чтобы потренироваться в:
- работе с WinForms
- взаимодействие с командой людей через GitHub
- работе наследований и переопределений состояний объекта

## 🧠 Как это устроено

Кратко про архитектуру:

- Реализован отдельный проект BallCommon отвечающий за рисование шарика, спавн, движение
- В каждой игре переопределяется по своему данный шарик в зависимости от правил игры

>  К сожалению на ранних этапах не знал как лучше сделать и в моём случае каждое движение шарика это - "нарисовать,стереть себя, переместить координату,снова нарисовать себя"
  из-за этого появляются мерцания + лаги при большой колличестве шаров на экране
